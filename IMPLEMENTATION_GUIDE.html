Step 3: Update Firestore Structure

Create these collections in Firebase Console → Firestore → Collections:

Collection: kasir

Fields:

{ nama: string, pin: string (4-6 digit), aktif: boolean, deposit: number, tglTambah: timestamp }

Collection: pengantar

Fields:

{ nama: string, hp: string, alamat: string, kendaraan: string, plat: string, deposit: number, metodeDeposit: string, fotoKtp: string (URL), fotoStnk: string (URL), fotoBukti: string (URL), status: string (pending/verified/approved/rejected), verified: boolean, tglDaftar: timestamp }

Collection: pesanan

Fields:

{ nama: string, hp: string, jenis: string, bayar: string, total: number, items: array, catatan: string, alamat: string, status: string, platformFee: number, tip: number, tglOrder: timestamp, noOrder: string, importedFromWA: boolean (if from fallback) }

Collection: admin

Fields:

{ username: string, password: string (⚠️ Use proper hashing in production!), tglTambah: timestamp }

⚠️ Security Warning: Currently using plain text passwords. For production, implement proper password hashing (bcrypt, argon2, etc.)

Step 4: Update Firestore Security Rules

Go to Firebase Console → Firestore Database → Rules tab and paste:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Public menu match /menu/{document=**} { allow read; } // Customer orders match /pesanan/{document=**} { allow create: if true; allow read, update: if resource.data.hp == request.auth.token.phone || request.auth.token.role == 'admin'; allow delete: if request.auth.token.role == 'admin'; } // Kasir accounts (admin only) match /kasir/{document=**} { allow read, write: if request.auth.token.role == 'admin'; } // Pengantar accounts match /pengantar/{document=**} { allow create: if true; allow read, update, delete: if request.auth.token.role == 'admin' || resource.data.hp == request.auth.token.phone; } // Admin match /admin/{document=**} { allow read, write: if request.auth.token.role == 'admin'; } } }
